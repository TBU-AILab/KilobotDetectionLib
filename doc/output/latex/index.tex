 Library containing basic functions and classes for Kilobot detection and tracking, camera calibration and distance measuring.\hypertarget{index_autotoc_md1}{}\doxysection{Implementation into your project}\label{index_autotoc_md1}
There are two simple ways how to implement the library into your project. The whole library project is based on CMake, so you can directly use it as a CMake subproject in your application. Because the library is composed of only a few files, the files can be copied into your project separately, and any build system can be used.\hypertarget{index_autotoc_md2}{}\doxysubsection{Using CMake subproject}\label{index_autotoc_md2}
The library can be cloned into your project structure as a subfolder. In that case, you need to append the following lines to your main CMake\+List.\+txt file.


\begin{DoxyCode}{0}
\DoxyCodeLine{  \# add subdirectory and enable the library compilaction}
\DoxyCodeLine{add\_subdirectory(Kilobot_detection_library)}
\DoxyCodeLine{}
\DoxyCodeLine{\# link library with your app and set include paths}
\DoxyCodeLine{\# NEED TO BE PLACED AFTER  add\_executable/add\_library command }
\DoxyCodeLine{target\_link\_libraries(\$\{CMAKE\_PROJECT\_NAME\} Kilobot_detection_library)}
\DoxyCodeLine{target\_include\_directories(\$\{CMAKE\_PROJECT\_NAME\} PUBLIC [path to the lib]/include)}

\end{DoxyCode}


Moreover, the library project also contains example applications. Their compilation is disabled by default. You can enable the compilation by adding the following -\/D parameters into your cmake command.


\begin{DoxyCode}{0}
\DoxyCodeLine{cmake -\/DBUILD\_EXAMPLES:BOOL=ON ../}

\end{DoxyCode}
\hypertarget{index_autotoc_md3}{}\doxysubsection{Using only library files}\label{index_autotoc_md3}
The library is composed of files placed in /include and /src folder. These files must be copied into your project. If you use a different folder structure, maybe the correction of include paths will be necessary.\hypertarget{index_autotoc_md4}{}\doxysection{Examples}\label{index_autotoc_md4}
The example applications are placed in "{}/examples/ folder. Each example in specific subfolder. Compilation of examples is disabled in default. If you want to enable it you need to append following -\/D parameter into your cmake command.


\begin{DoxyCode}{0}
\DoxyCodeLine{cmake  -\/DBUILD\_EXAMPLES:BOOL=ON ../}

\end{DoxyCode}
